## cyber security course

#### Goals: Have a eagel eye view of the current cyber security landscape and basic understanding of all topics that an IT manager/leader needs to know.

#### Outline/Agenda: 
 1. Threat modeling
 2. Encryption concepts
 3. Authentication and hashes
 4. Network scanning
 5. Firewall
 6. Exploit development , Network pentesting and introduction to tools that are used by testers 
 7. Cloud Security and threat detection (AWS GuardDuty) 
 8. Compliance (ISO 27001)
 9. Tools
    * Kali Linux
    * Zap scan
    * Burp Suite
    * Nmap

#### Exercises:

1. Create a Simple Risk Register
2. Boot Kali in VM.
3. Run a simple port scan
4. Explore tools
   * nmap
   * hydra
   * metasploit-framework
   * nikto
6. Encrypt & Decrypt
7. Crack Weak Hashes Using Hashcat
8. port scanning
9. understand firewall rules and go through WAF rules in AWS
10. learn GuardDuty , its usage and examples
11. Building end to end cyber security solution for an IT firm


#### Models of frameworks
1. NIST Cybersecurity Framework
2. ISO 27005 Risk Management
3. MITRE ATT&CK

#### Visiting lecturers
1. Real heackers
2. Entrepreneurs in this field like Risto Siilasmaa and Petri Allas 
Rationale:
Learning how major companies detect and respond to attacks would prepare me to design more secure, resilient solutions.

#### Excursions:
Visit a data centre.

#### Think outside the box
I would create a course that would blend technical depth with real-world exposure, preparing me not just for exams, but for the evolving cybersecurity landscape that is rapidly becoming more complex and more exciting.
